# サッカートラッキングデータと言語を統合した大局的戦術マルチモーダル質問応答モデルの提案

## 1. 研究背景（Research Background）

### 1.1 大規模言語モデルとマルチモーダルAIのパラダイムシフト
近年、自然言語処理（NLP）の分野において、トランスフォーマー（Transformer）アーキテクチャを基盤とした大規模言語モデル（Large Language Models: LLMs）が驚異的な発展を遂げている。特に、大量のWebテキストを用いた事前学習（Pre-training）の後に、多様なタスクを自然言語の指示（命令）と回答のペアに変換して追加学習を行う「指示チューニング（Instruction Tuning）」や、人間のフィードバックに基づくアライメント（SFT, DPO等）の導入により、LLMは未知の命令に対しても柔軟に従う高度な汎化性能を獲得した。

この言語ドメインにおけるブレイクスルーは、画像や動画といった非言語モダリティを統合する「マルチモーダル大型言語モデル（Multimodal Large Language Models: MLLMs）」へと急速に拡張されている。その先駆例であるLLaVA（Large Language and Vision Assistant）等に代表されるアプローチでは、強力な視覚エンコーダー（CLIP等）によって抽出された高次元の視覚特徴量を、線形投影層（Linear Projection）やQ-Formerといった軽量な接続層（ハブ）を介して、LLMのトークン埋め込み空間へとマッピング（接地: Grounding）させる。これにより、AIは単に画像内のオブジェクトを認識・分類する知覚タスク（Perception Task）を超え、人間の自由な問いかけに対して視覚的エビデンスに基づきながら論理の連鎖（Chain-of-Thought）を踏まえて回答する、双方向のマルチモーダル質問応答（Multimodal Question Answering: Multimodal QA）を可能にした。さらに、近年のLlama-3-8Bモデルに代表される高密度な小規模オープンソースモデルの台頭により、膨大なトークン数の事前学習を経ることで、比較的少ないパラメータ数でありながら強力な命令追従能を保持する実用的なデコーダーの利用環境が整備されている。

### 1.2 スポーツアナリティクスにおけるクロスモーダル理解の現状
このマルチモーダルAIの技術的潮流は、スポーツアナリティクス、とりわけ世界的に競技人口およびデータ密度が高いサッカーのドメインにおいても反映され始めている。従来のサッカーAI研究は、オブジェクト検出やポーズ推定を用いた選手・ボールのトラッキング、あるいはディープラーニングを用いたパスやシュートといった特定イベントの単一分類（タスク固有モデルの構築）が主であった。

しかし近年、マルチモーダル基盤モデルの概念をサッカー映像に適用し、試合動画を入力として、タイムラインに沿った自動実況テキストやイベント解説を生成するクロスモーダルな「映像-言語変換モデル」（UniSoccer等）の研究が提案され、大きな注目を集めている。これらの研究は、映像内で発生した局所的なイベントを言語空間へと接続（アラインメント）させる上で多大な功績を残した。しかし、これらの既存研究の多くは、あらかじめ定義されたイベントやシーンに対する「映像からテキストへの単方向的な記述（Video Captioning）」の枠組みに留まっている。サッカーの現場における指導者、アナリスト、あるいは熱狂的なファンが求める真の知能的支援とは、単なる「起きた事実の機械的な実況」ではなく、「なぜその戦術が選択されたのか」「その時の守備布陣の弱点はどこにあったのか」といった、ユーザーからの複雑かつ多様な問いかけ（指示テキスト）に対し、状況に応じた柔軟な解説を行う「双方向の対話型マルチモーダル戦術QA」である。したがって、サッカーAI研究における次の決定的なパラダイムシフトとして、高度なマルチモーダルQAモデルの確立が強く求められている。

## 2. 技術的課題（Research Challenges）

サッカーという極めて複雑で流動的なチームスポーツにおいて、フォーメーションの推定、守備ラインの高さの算出、攻守の戦術意図の解釈といった「大局的な戦術推論QA」を解くマルチモーダルモデルを構築しようとする場合、従来の映像ベースのアプローチでは、構造的に突破不可能な以下の3つの致命的な技術的障壁（Research Challenges）に直面する。

### 2.1 記述言語の局所性と物理データの広域性のミスマッチ（ミクロとマクロのギャップ）
第1の課題は、サッカーの試合に付随する天然の言語資産（放送の実況文や解説テキスト）が抱える認知バイアスに起因する。人間の実況者や解説者が発する言葉は、認知心理学的な制約から、常にボール保持者やゴール前の決定的な攻防といった**「ミクロなイベント意味論」**に著しく偏向（バイアス）している。
しかし、サッカーにおける戦術の本質は、ボール周辺の動向だけでなく、ボールを保持していない残りの20人以上の選手が織りなすチーム全体の布陣の変形、スペースの創出、あるいはバックラインの統率といった、ピッチ全体（グローバル）の**「マクロな空間幾何構造」**によって規定される。例えば、「守備側がコンパクトなミドルブロックを形成して中央を閉鎖している」というマクロな戦術状態は、個々のパスやトラップといったミクロな単一イベントの記述からは逆算できない。既存のイベント記述テキストをそのまま教師信号（キャプション）としてマルチモーダル学習に流用しようとしても、そこにはマクロな戦術幾何構造を言語空間へと接地させるための直接的なシグナル（対応関係）が含まれておらず、これがモデルに大局的な戦術概念を理解させる上での致命的な「ドメイン間ギャップ」となっていた。

### 2.2 映像モダリティ特有の幾何学的・物理的制限と情報ボトルネック
第2の課題は、入力モダリティとして「映像（動画の画素テンソル）」を採用すること自体の構造的限界である。サッカーのテレビ中継映像や一般的なカメラワークは、視聴者の視認性を確保するためにボール周辺をクローズアップ（拡大）する。この画角制限（放送フレームの制約）により、画面外（アウト・オブ・ビュー）で生じている戦術的に極めて重要な挙動（例：相手のカウンターを警戒してラインをコントロールするディフェンダーの動きや、逆サイドで幅を取ってパスを呼び込むウイングのポジショニング）は、データそのものが物理的に欠落する。また、複数の選手が密集するエリアでは、カメラの視点に起因する選手同士の重なり（オクルージョン）によって、各選手の位置関係を正確に識別することが困難となる。

さらに、マルチモーダルAIが動画を処理する際の数理的な制約も大きい。1フレームの動画をLLMへ流し込むだけで数百トークンもの視覚トークンが消費されるため、高密度の時系列（30fps等）をそのまま入力すると、LLMのコンテキスト窓のパンクや計算コストの爆発（トークン爆発）を引き起こす。これを回避するため、現代の主要なVideo-LLM（あるいはUniSoccer等のモデル）は、Q-Formerや時間・空間方向のダウンサンプリング、あるいはフレームの大幅な間引き（サンプリングの粗雑化）によって、動画データを極少数（例：30トークン等）の視覚トークンに強力に圧縮せざるを得ない。この過度な情報の丸め込み（圧縮プロセス）において、選手22人のミリメートル単位の精密な相対位置関係（トポロジー）のニュアンスは完全に破壊され、平均化されて混ざり合ってしまう。結果として、イベント（ゴール等）のような強烈な視覚적変化は捉えられても、フォーメーションやDFラインの高さといった「緻密な幾何学バランス」をLLMが処理するための空間解像度は物理的に消滅するという、致命的な情報ボトルネックが存在していた。

### 2.3 トラッキングドメインにおけるアノテーション（キャプション）データの不在
上記2点の限界から、ピッチ全体の完全かつ厳密なマクロ物理情報を1/25秒単位のミリ秒精度で常に保持している「サッカートラッキングデータ（全22選手およびボールの2次元座標時系列テンソル）」こそが、大局的な幾何戦術推論QAを行うための最適かつ本質的な非言語モダリティであることは明白である。トラッキングデータであれば、画面外の情報の欠落もオクルージョンも存在せず、データサイズ自体も動画の数万分の1と極めて軽量であるため、情報を間引くことなくLLMへ届ける器（ポテンシャル）がある。

しかし、ここに第3の、核心的な壁が立ちはだかる。画像ドメイン（COCO等）や動画ドメインにはインターネット由来の大規模な「画像/動画 ＋ キャプション」のペアデータセットが豊富に存在するが、トラッキングデータは純粋な数値時系列（`.npy` や `.csv` 形式の座標の羅列）であり、それに対応する詳細な戦術記述やQAが施された自然言語コーパス（キャプションデータセット）が地球上に構造的に存在しない。この「キャプションデータの絶対的な欠如（Data Scarcity）」に起因して、トラッキングデータと言語セマンティクス（意味論）をあらかじめ強力にバインドさせた「サッカー版CLIP」のような事前学習済みエンコーダーを構築することができず、これがこれまでトラッキングデータを用いたマルチモーダルQA研究の着手を根底から阻んできた本質的な因果関係であった。

## 3. 研究目的（Research Objectives）

本研究の主たる目的は、既存の映像ベースのクロスモーダル研究が抱えていた幾何学的限界と、天然の言語記述が持つ局所性（ミクロバイアス）の壁を同時に打破し、サッカートラッキングデータと自然言語指示を同一のアーキテクチャ内で直接統合する、世界初の**「大局防衛・攻撃サッカー戦術マルチモーダル質問応答（QA）モデル」**を構築・提示することである。

具体的には、単に「座標からイベントを当てる」という知覚レベルのタスクを超え、数値幾何学（物理量）と言語概念（戦術意味論）の巨大なセマンティック・ギャップを架橋し、以下の3つの具体的なサブゴールを達成することを目指す。

### 3.1 空間トポロジーを破壊しない時空間マルチモーダル表現の獲得
時系列予測（軌跡回帰タスク等）を課すことで、選手・ボールの物理的な運動ダイナミクス（速度・加速度・相対距離の変化）を事前に獲得させたトラッキングエンコーダーを構築する。このエンコーダーから出力される高次元幾何特徴量を、LLMへ渡す段階で一律に平均化（mean pooling）して丸め込む既存のボトルネックを排除し、22人の個別の空間配置構造（トポロジー）およびチームごとの布陣関係を完全に保持したまま、LLM（Llama-3-8B-Instruct等）のデコーダーが認識できる言語トークン空間へと高効率にマッピングさせる接続層（Linear Projection、あるいは空間位置補正付きQ-Former）のアーキテクチャを確立する。

### 3.2 ミクロ（イベント）とマクロ（戦術幾何）を統合した対話型推論の実現
トラッキングデータにおけるキャプションデータセットの不在（2.3の課題）を解消するため、LLaVAが画像ドメインで成功させたデータ合成パイプライン（Visual Instruction TuningにおけるGPT-4活用）の精神をトラッキング空間へと拡張・適応する。
生のトラッキングテンソルから、数値計算および幾何学的ルールベース（K-Meansを用いた1次元クラスタリングによるライン抽出、重心計算によるコンパクト性算出等）によって、「フォーメーション」「守備ラインの高さ」といった**『マクロな戦術幾何ファクト（中間表現）』**を厳密に抽出する。このハードな真値（ファクト）を、既存の「ミクロなイベント（パス、ゾーン、プレッシャー等）」のメタデータと共にテキスト専用のLLM（GPT-4o）へ仲介させ、時空間においてミクロとマクロが完全に同期した、ハルシネーション（それっぽい嘘）のない高品質な戦術指示追従データ（SFT用自由記述QAデータセット）を自律生成するパイプラインを構築する。これをモデルに学習（指示チューニング）させることで、LLMの持つ高度なサッカー知識をフックにしながら、「なぜその布陣をとったのか」「その時ラインはどう動いたか」という高度な戦術前後の因果関係を論理的に解説できる対話知能を獲得させる。

### 3.3 将来的な「戦術・選手個人評価」へ拡張可能な基盤フレームワークの提示
局所的なアクションの成否（パスが成功したか否か）を記述する従来の評価レイヤーから脱却し、チーム全体の戦術構造の動的変化（マクロ）と、その構造内における選手個人のポジショニングやスプリント（ミクロ）の相互作用を、LLM内部の特徴量アテンションによってダイレクトにハンドリング可能にする。これにより、将来的に「このプレーにおいて、戦術的・スペース的に最も効果的なフリーランニングで相手を破綻させた選手は誰か」「最も致命的な守備ラインの乱れを生んだ選手は誰か」といった、プロのアナリストレベルの高度な個人評価および戦術スカウティングタスクへとシームレスに拡張可能な、新しいスポーツマルチモーダルAIの強力な基盤フレームワーク（基盤プラットフォーム）を社会および学術界に提示する。
