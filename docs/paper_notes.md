# 論文執筆ノート

## Grounding Problem の学術的位置付け

### 課題の名称
「Language Prior Dominance」または「Modal Collapse」として既知のホットな問題。
本研究では「△Bパターン」（指示形式には従うがトラッキングデータを無視して幻覚を生成）として実験的に観測・命名した。

### なぜトラッキングデータは映像より難しいか

| 観点 | 映像 | トラッキングデータ |
|------|------|-------------------|
| 意味密度 (Semantic Density) | 背景・色・質感など LLM の知識と紐付きやすいヒントが豊富 | 座標数値という極めて抽象的な情報 |
| 事前アライメント | CLIP 等で数億規模の画像-テキストペアで学習済み | CLIP 相当物が存在せず、ゼロから構築が必要 |
| 空間時系列粒度 | 一目でわかる「プレス」も | 22人の多次元時系列相関として解釈必要 |

**教授への一言回答（「映像でいいのでは？」に対して）**:
映像には CLIP という巨大な橋がある。トラッキングデータにはない。
この「ゼロからのアライメント問題」を解くこと自体が本研究の課題設定。

---

## 関連研究との位置付け

- **UniSoccer (2024)**: サッカー特化の先行研究。アノテーション品質とアライメントの困難さを認めているが解決はしていない
- **Time-LLM (2023/24)**: 一般時系列データの "Reprogramming" で時系列パッチを LLM のテキスト埋め込み空間にマッピング。本研究との違いは「スポーツの戦術的意味論」に特化した点
- **Video-LLaVA**: "Alignment before projection" の重要性を示す。本研究の Q-Former + Alignment Loss の設計根拠と共鳴

**結論**: 「誰もやっていない」ではなく、「汎用手法をそのまま持ってくるだけではサッカーのような専門的・低密度データで Grounding が崩壊する」という課題設定が出発点。

## Contribution の整理（論文骨子案）

**C1 Problem Identification**
サッカーのトラッキングデータを用いた自然言語 QA において、LLM の事前知識がモダリティ入力を凌駕し、事実に基づかない回答（△Bパターン）を生成する Grounding 問題を実験的に特定した。

**C2 Architecture**
23人の選手の時空間相関を捉える専用の TrackingEncoder と、情報を凝縮する Q-Former を組み合わせた End-to-End パイプラインの提案。

**C3 Proposed Method（Token-Word Alignment）**
少量データでの Grounding を成功させるため、Q-Former のトークンを LLM の語彙空間へ意味的に誘導する Slot-based Token-Word Alignment 手法の提案。

**C4 Instruction Diversification**
少量データ環境での汎化のため、指示文多様化（6バリエーション）により LLM の Instruction Following 能力を引き出す手法を示した。

**C5 Experimental Results**
提案手法により、LLM が指示文のエコーを脱し、生のトラッキングデータに基づいた戦術解説が可能になったことを定量的・定性的に示した。

## 損失関数の定式化

### 全体損失
$$\mathcal{L}_{total} = \mathcal{L}_{LM} + \lambda_{align} \mathcal{L}_{align} + \lambda_{slot} \mathcal{L}_{slot}$$

### Slot-based Token-Word Alignment Loss（実装準拠・コサイン類似度版）
$$\mathcal{L}_{slot} = \sum_{s=0}^{2} \left(1 - \cos\left(\text{proj}_s\!\left(\bar{q}_s\right),\ \overline{\text{embed}(w_s^{gt})}\right)\right)$$

- $\bar{q}_s$: スロット $s$ のトークン平均（Q-Former 出力）
- $w_s^{gt}$: スロット $s$ の GT ラベルテキスト（action / zone / pressure）
- $\text{proj}_s$: スロット固有の線形投影層

> **注意**: Gemini の数式は L2 ノルム版だが、実装はコサイン類似度。スケール不変性の点でコサイン版が適切。論文はコサイン版で統一する。

### Slot 割り当て
| Slot | トークン | 対応ラベル |
|------|---------|-----------|
| 0 | 0–7 | action |
| 1 | 8–15 | zone |
| 2 | 16–23 | pressure |
| 3 | 24–31 | summary（align なし・生成損失のみ） |

## 学術的文脈での説明フレーム

「単に繋いでみた」ではなく：

> 「既存の VLM 手法（BLIP-2 / Q-Former）を、事前学習データのない特殊モダリティ（サッカートラッキング）に適用した際、LLM の言語バイアスが勝ってしまう問題（△Bパターン）を特定し、それを解決するスロット別アライメント手法を提案した」

これは **表現学習とクロスモーダル・アライメントの設計論** として位置付けられる。

## 【重要】アプローチ2の前提条件に関する考察

> **氏原の指摘（Geminiの誤りを修正）**

BLIP-2以降が「アプローチ2（アライメント特化）に近い」というのは**誤り**。前提が根本的に違う：

- **BLIP-2 / LLaVA**: CLIPが数億枚の画像で「意味」を習得済み → Encoderの出力が言語空間に「半分足を突っ込んでいる」状態 → Q-Formerはその橋渡しに専念できる
- **本研究**: TrackingEncoderは軌跡予測という物理タスクで学習 → 出力は「座標の幾何学的特徴」のみ → 言語構造と1mmもリンクしていない

### Semantic Gap（意味の溝）の深さ

| | 画像系（BLIP-2等） | 本研究（トラッキング） |
|--|--|--|
| Encoderの出力空間 | 視覚的・意味的（言語空間に近い） | 物理的・幾何学的（言語と無関係） |
| CLIPの有無 | あり（数億ペアで学習済み） | なし |
| Q-Formerの役割 | 意味空間の圧縮・橋渡し | 意味のない信号を意味空間に変換 |

**核心**: 「どれだけ中間層（Q-Former）を賢くしても、Encoderの出力に言語情報がなければ砂の上に城を建てるようなもの」

### 2つの解決策

**案A（Encoder改造）**: Encoder学習時にラベルとの対照学習（Contrastive Learning）を組み込み、「サッカー・セマンティック・エンコーダー」を構築する

**案B（逆伝播による蒸留）**: アライメント損失の勾配をQ-Former経由でEncoderまで逆伝播させ、LLMの埋め込み空間の構造をEncoder側に蒸留する → **現在のB-2実装はここに相当**

### 教授への説明フレーム（改訂版）

> 「既存の映像LLMはCLIP等のおかげで意味の抽出を他力本願にできています。しかしトラッキングデータにはその基盤がありません。本研究の真の課題は、**『ただの座標の羅列（物理量）から、LLMが対話可能なレベルのセマンティクスを抽出するエンコーダーをどう構築するか』**です。Q-Formerを橋渡しではなく、**エンコーダーに言語構造を逆流させるフィルター**として機能させます。」

### 論文ストーリーの再構成

1. **課題**: トラッキングデータにはCLIPのような事前学習済み意味空間が存在しない
2. **失敗の証明**: 単純にBLIP-2構造を適用しても、EncoderにセマンティクスがないためLLMは入力を無視する（△Bパターン）
3. **提案**: アライメント損失の勾配をEncoder側まで通すことでEncoder自体を言語空間に引き寄せる（案B）
